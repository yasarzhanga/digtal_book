# 数字教材平台沉浸式展示 Demo V2：产品规格

## 0. 产品定位

本项目不是管理后台，也不是把纸质教材内容搬到网页上。它要通过一个可演示、可操作、可持久化的完整样章证明数字教材相对“纸质教材 + 二维码/外部资源链接”的五个核心优势：

1. **内容与资源一体化**：文字、公式、图片、音频、视频、3D、全景、图表、仿真实验和练习在同一阅读流中完成，不跳转到外部平台。
2. **知识可以操作**：学习者可拖动参数、旋转模型、查看热点、运行仿真，而不是只观看静态图。
3. **学练测即时闭环**：概念讲解、媒体观察、交互实验、随堂练习和反馈位于同一章节。
4. **学习过程可记录**：阅读、媒体播放、仿真实验、答题、笔记和录音都形成真实行为数据。
5. **教学过程可联动**：教师可同步定位、推送当前练习，并实时看到课堂与课后学习数据。

所有 P0 功能必须可真实操作、写入 SQLite、刷新后仍存在。不可用静态页面、无行为按钮、假进度条或写死图表冒充功能。

## 1. 演示主题与主线

示范教材固定为：

> **《大学物理：牛顿第二定律实验课》**

用一套内容完成编辑者、学生、教师三端演示。

### 1.1 三个章节

#### 第一章：从纸面概念到多媒体观察

包含：

- 完整富文本排版；
- 行内知识气泡；
- 带热点的受力示意图；
- 三张实验过程画廊；
- 牛顿第二定律音频讲解与同步文字稿；
- 小车受力实验视频、字幕和视频文字稿；
- 扩展阅读侧边抽屉。

#### 第二章：把公式变成可操作实验

包含：

- 行内公式和块级公式；
- 公式参数代入演示；
- 可旋转缩放的 3D 小车模型；
- 360° 实验室全景；
- `F = ma` 交互仿真实验；
- 可交互折线图；
- 关键结论提示块。

#### 第三章：学、练、测、表达一体化

包含：

- 单选、多选、判断、填空四种题型组成的题组；
- 即时判分、解析与重做；
- “用自己的话解释牛顿第二定律”录音任务；
- 实验指导书 PDF 在线预览；
- 力、质量、加速度、牛顿第二定律之间的可交互知识图谱；
- 章节总结和个人学习轨迹。

### 1.2 传统教材对比模式

阅读器顶部必须提供：

```text
传统教材视图 | 数字教材视图
```

两种模式读取同一发布版本：

- **传统教材视图**：只保留文字、静态缩略图和“资源二维码/外链卡片”的表现，不允许直接播放和操作；
- **数字教材视图**：完整呈现所有内嵌资源与交互组件。

切换时保持当前章节与滚动位置。页面右侧显示五个价值标签：

```text
内容内嵌
知识可操作
即时反馈
学习留痕
教学可见
```

此模式是 Demo 的核心讲解入口，不是装饰页。

## 2. 演示角色与快捷入口

### 2.1 角色

- **编辑者**：创作、导入、插入富媒体、预览、发布；
- **教师**：开始课堂、同步位置、推送题目、签到、查看数据；
- **学生**：阅读、操作媒体、答题、标注、录音、同步教师位置。

### 2.2 演示账号

```text
editor@demo.local / demo123456
teacher@demo.local / demo123456
student@demo.local / demo123456
```

另创建 7 名数据库中的示例学生，并生成历史活动记录，用于展示有意义的班级统计。统计必须从真实表聚合，禁止在前端写死。

### 2.3 演示模式

首页 `/demo` 提供三个“一键进入”按钮。仅在 `DEMO_MODE=true` 时显示，可由服务端创建对应 Session。正常登录页仍然存在。

顶栏提供演示角色切换器，便于现场在编辑者、学生、教师之间切换。切换必须真正更换服务端 Session，不是在前端假装角色。

## 3. P0 功能范围

## 3.1 内容创作工作台

路由：

```text
/editor/books
/editor/books/[bookId]
/editor/books/[bookId]/preview
/editor/books/[bookId]/assets
/editor/books/[bookId]/versions
```

桌面端布局：

- 左栏：教材信息、章节树、目录拖拽；
- 中间：真正的所见即所得编辑器；
- 右栏：当前节点属性、组件配置和资源选择；
- 顶部：撤销/重做、保存状态、设备预览、版本发布。

手机端不要求完整编辑，只显示只读预览和“请使用桌面端创作”；平板和电脑端必须可编辑。

### 3.1.1 真正的富文本编辑器

必须使用 TipTap 或同等级成熟编辑器，不允许自制一个简单 `contenteditable` 代替。

工具栏必须可见、可操作并对当前选区生效：

- 段落、H1-H6；
- 字体、字号；
- 加粗、斜体、下划线、中划线；
- 字体颜色、背景高亮色；
- 上标、下标、行内代码；
- 左/中/右/两端对齐；
- 行高；
- 增加缩进、减少缩进；
- 无序列表、有序列表、任务列表；
- 引用、代码块、分割线；
- 链接；
- 清除格式；
- 撤销、重做；
- 查找、替换；
- Markdown 快捷输入；
- 斜杠命令菜单；
- 插入表格、图片、公式和全部富媒体组件。

至少实现以下键盘体验：

- `Ctrl/Cmd + B/I/U`；
- `Ctrl/Cmd + K`；
- Markdown 标题、列表、引用快捷输入；
- `/` 打开组件菜单；
- `Ctrl/Cmd + Z` 和 `Shift + Ctrl/Cmd + Z`。

### 3.1.2 表格

支持：

- 新建表格；
- 增删行列；
- 合并/拆分单元格；
- 表头行；
- 文本对齐；
- 单元格背景色；
- 列等宽；
- 自适应宽度；
- 删除表格；
- 从 Excel/网页复制并粘贴二维表格数据。

复杂斜线表头不进入 P0。

### 3.1.3 章节与文档操作

- 一级、二级章节；
- 新增、重命名、删除；
- 同级拖拽排序；
- 二级章节可跨一级章节拖动；
- 编辑器中的块级节点可通过拖拽手柄排序；
- 节点可复制、删除；
- 自动从 H1-H6 生成节内大纲；
- 点击大纲定位；
- 统计字数和各组件数量。

### 3.1.4 资源库

资源库不是简单上传框，至少包含：

- 网格/列表视图；
- 图片、音频、视频、3D、全景、PDF/附件分类；
- 文件名、类型、大小、时长或尺寸；
- 搜索；
- 上传；
- 删除未引用资源；
- 显示被哪些章节引用；
- 从资源库拖入或插入到编辑器当前位置；
- 相同资源可重复引用，不重复存储。

文件限制：

- 图片：PNG/JPEG/WebP/SVG，10MB；
- 音频：MP3/WAV/OGG，30MB；
- 视频：MP4/WebM，100MB；
- 3D：GLB/GLTF，50MB；
- 全景：2:1 JPEG/WebP，20MB；
- 附件：PDF/DOCX/XLSX/PPTX，50MB。

文件保存在 `storage/uploads`，文件名随机化，读取使用受控路由，防目录穿越。

### 3.1.5 DOCX 导入

支持 `.docx`：

- Heading 1/2 转一级、二级章节；
- Heading 3-6 保留为正文标题；
- 加粗、斜体、下划线、列表、链接、图片、表格；
- 段落转 TipTap 节点；
- 导入后继续编辑；
- 导入预览中显示“将创建的章节、媒体和表格数量”；
- 用户确认后写入草稿；
- 导入失败给出明确原因。

包内提供 `starter-assets/imports/sample-physics.docx` 作为测试夹具。

### 3.1.6 保存、离线恢复与发布

- 停止编辑约 800ms 后自动保存；
- 顶栏明确显示“正在保存 / 已保存 / 本地备份 / 保存失败”；
- 同时将最近文档 JSON 写入 IndexedDB 或 localStorage；
- 服务端失败时保留本地版本；
- 再次进入如本地版本更新，提示比较和恢复；
- 发布使用数据库事务生成不可变快照；
- 发布后版本号递增；
- 阅读器只读已发布快照；
- 版本页可查看发布时间、组件数量和发布备注；
- 可将任意历史版本设为当前阅读版本，但不覆盖草稿。

### 3.1.7 多端预览

编辑器内预览支持：

```text
电脑 1440
平板 834
手机 390
```

预览必须使用与正式阅读器相同的渲染组件，不能另做一套静态假页面。

## 3.2 内容组件与富媒体能力

所有组件必须同时具备：

1. 编辑器中的可视节点；

5. 操作事件记录；

阅读能力：

阅读能力：

### 3.2.5 公式

- 行内公式；
- 块级公式；
- LaTeX 输入；
- 实时 KaTeX 预览；

- 单系列颜色配置；

阅读能力：

- 悬停数值；

### 3.2.7 `F = ma` 交互仿真实验

这是 Demo 中最重要的数字化组件，必须是真实计算，不是动画 GIF。

编辑器属性：

- 力的最小值、最大值、步长、默认值；
- 质量最小值、最大值、步长、默认值；
- 是否显示轨迹图；
- 是否显示实时公式；
- 引导问题文本。

学生端：

- 力和质量滑块；
- 开始、暂停、重置；
- 小车根据 `a = F / m` 真实计算并动画；
- 实时显示 `F`、`m`、`a`；
- 自动绘制速度—时间或位移—时间曲线；
- 可固定质量比较不同力，也可固定力比较不同质量；
- 一键保存本次实验数据；
- 保存后出现在个人实验记录中；
- 每次运行和参数变化写入活动事件。

## 3.3 数字教材阅读器

路由：

```text
/reader/books/[bookId]
/reader/books/[bookId]/resources
/reader/books/[bookId]/notes
/reader/books/[bookId]/experiments
/reader/books/[bookId]/report
```

### 3.3.1 布局和阅读设置

桌面：

- 左：目录、目录图谱、全文搜索；
- 中：混合媒体正文；
- 右：大纲、笔记、资源、学习轨迹。

手机：

- 两侧栏变为抽屉；
- 媒体组件占满可用宽度；
- 视频、3D、全景、图表和仿真支持触摸；
- 到章节底部可进入下一章。

阅读设置：

- 字号；
- 行高；
- 内容宽度；
- 浅色、护眼、深色；
- 专注模式；
- 上一节、下一节。

### 3.3.2 搜索

- 全书正文检索；
- 资源标题和描述检索；
- 结果显示章节和上下文；
- 点击精准跳转到节点；
- 搜索结果区分正文、图片、音频、视频、3D、全景、附件、题目。

### 3.3.3 资源中心

自动聚合发布版本中的所有资源：

- 图片；
- 画廊；
- 音频；
- 视频；
- 3D；
- 全景；
- 图表；
- 仿真；
- 附件；
- 题组。

支持按类型筛选、搜索、点击跳回原文。不得维护第二份人工资源列表。

### 3.3.4 标注、笔记与朗读

- 在同一富文本块内选择任意文字；
- 黄色、绿色、蓝色、粉色高亮；
- 添加笔记；
- 保存时间和位置；
- 笔记列表可跨章节跳回；
- 可编辑和删除；
- 选中文字后可调用浏览器 `SpeechSynthesis` 朗读；
- 中英文根据文本自动选择可用语音；
- 不声称使用云端 AI。

### 3.3.5 阅读记忆和学习轨迹

记录：

- 上次章节和节点；
- 已访问章节；
- 正文有效阅读时长；
- 各音视频播放时长和完成率；
- 图表查看；
- 3D/全景交互；
- 仿真实验运行与保存次数；
- 答题记录；
- 笔记数量；
- 录音任务提交。

页面前台且可见时才累计阅读时长。事件批量上报到数据库。

学习轨迹按时间倒序显示，例如：

```text
10:21 完成小车实验，F=6N，m=2kg，a=3m/s²
10:18 视频观看至 92%
10:15 打开“惯性”知识气泡
10:12 添加一条蓝色笔记
```

### 3.3.6 个人学习报告

必须来自数据库真实记录：

- 总阅读时长；
- 章节进度；
- 音频/视频完成率；
- 3D、全景和仿真交互次数；
- 实验保存记录；
- 题组提交次数、最近得分、正确率；
- 笔记数；
- 录音提交情况；
- 学习活动趋势。

## 3.4 教学端

路由：

```text
/teacher/courses
/teacher/classes/[classroomId]
/teacher/classes/[classroomId]/live
/teacher/classes/[classroomId]/analytics
/teacher/classes/[classroomId]/students/[studentId]
```

### 3.4.1 课程和班级

- 从已发布教材创建课程；
- 创建班级；
- 6 位邀请码；
- 加入二维码；
- 学生用邀请码加入；
- 重复加入幂等。

种子数据默认已有：

```text
课程：大学物理示范课
班级：物理 1 班
邀请码：PHYS01
```

### 3.4.2 师生同步定位

- 教师开始课堂；
- 教师阅读同一发布版本；
- 任一内容节点可“设为当前教学位置”；
- 学生端显示当前课堂状态；
- 学生点击“同步教师位置”跳转；
- 教师切换位置后学生可再次同步；
- 课堂结束后状态明确变化。

使用 2—3 秒轮询即可，不要求 WebSocket。

### 3.4.3 随堂题推送

- 教师可从当前章节选择一个题组中的题目；
- 点击“发起随堂题”；
- 学生端出现非阻断提示并可进入答题；
- 学生提交后教师端实时刷新：
  - 已答人数；
  - 正确人数；
  - 选项分布；
- 结果必须来自 `LiveQuizResponse` 表；
- 教师可结束本题并展示正确答案。

### 3.4.4 签到

- 教师发起 5 分钟签到码；
- 学生输入签到码；
- 防重复；
- 教师查看已签到、未签到；
- 可手动改为出勤、请假、缺勤；
- 记录写数据库。

### 3.4.5 班级数据

种子数据含 8 名学生的历史真实记录，当前操作会继续累积。

班级总览：

- 平均阅读进度；
- 平均有效学习时长；
- 音频平均完成率；
- 视频平均完成率；
- 仿真实验参与率；
- 3D/全景参与人数；
- 平均答题正确率；
- 笔记和录音提交数；
- 最近 7 天活动趋势；
- 当前课堂在线与随堂题结果。

学生详情：

- 章节进度；
- 媒体完成情况；
- 实验记录；
- 答题详情；
- 笔记数；
- 录音提交；
- 活动时间线。

所有图表必须调用聚合服务读取数据库，不得在组件内写静态数组。

## 4. 数据与发布原则

### 4.1 同一内容引擎

编辑预览、正式阅读、教师课堂阅读必须共享同一套节点 Schema 和 Renderer：

```text
content-engine/schema
content-engine/editor
content-engine/renderer
content-engine/tracking
```

禁止为演示分别维护三套 HTML。

### 4.2 草稿与发布版本隔离

- 草稿使用 TipTap JSON；
- 发布生成完整不可变快照；
- 快照包含章节、文档 JSON、资源清单和版本信息；
- 阅读、标注、学习数据绑定具体发布版本；
- 草稿修改不得立即影响学生。

### 4.3 真实行为事件

至少记录：

```text
PAGE_VIEW

PANORAMA_HOTSPOT_OPEN
```

高频事件客户端批量上报，服务端校验后写入。

## 7. 视觉与交互要求

- 面向教育产品，不做普通 SaaS 管理后台风格；
- 正文阅读区重视排版和内容层次；
- 富媒体节点有统一边框、标题、说明、全屏和状态反馈；
- 数字教材模式中媒体自然嵌入正文，不呈现为一排外链卡片；
- 页面首次进入即可看到至少一个正在工作的交互组件；
- 重要演示入口不得藏在三级菜单；
- 所有图标使用统一图标库；
- 390px、834px、1440px 均需检查；
- 加载、空状态、错误状态、权限拒绝状态均需设计。

## 8. 8—12 分钟现场演示脚本

1. 进入 `/demo`，用一句话说明“五个数字化优势”；
2. 一键进入编辑者；
3. 在富文本中修改标题，演示颜色、上标、公式和表格；
4. 输入 `/`，插入视频或 3D 组件，从资源库选择本地素材；
5. 打开右侧属性面板，修改图注、字幕或热点；
6. 切换手机预览，确认同一内容自适应；
7. 发布新版本；
8. 切到学生，打开传统教材视图，展示资源只能作为外链/二维码；
9. 切到数字教材视图，依次播放音频/视频、旋转 3D、拖动仿真参数；
10. 保存一次实验，回答题组，添加笔记并提交录音；
11. 切到教师，开始课堂并把位置设到仿真实验；
12. 学生点击同步，跳转到同一组件；
13. 教师推送随堂题，学生作答，教师查看选项分布；
14. 教师打开班级统计，看到媒体、实验、答题和刚才课堂行为的真实数据。

## 9. 完成定义

只有以下全部满足，Codex 才能结束目标：

- 富文本工具栏不是占位，所有列出的 P0 操作有真实效果；
- 15 类核心富媒体/交互组件在编辑端可配置、阅读端可操作、发布后可恢复；
- 音频和视频使用本地素材可播放；
- 3D、本地全景和仿真实验真实可操作；
- 同一发布内容可在传统/数字两种模式切换；
- 学生行为写入数据库并进入个人、班级报告；
- 教师同步定位、随堂题和签到可跨角色完成；
- 核心数据刷新后存在；
- `npm run db:reset` 可恢复完整演示数据；
- lint、类型检查、单测、E2E 和生产构建全部成功；
- Playwright 覆盖编辑发布、数字阅读、媒体交互、仿真、答题、课堂同步、随堂题与统计；
- 生成 1440px、834px、390px 的关键页面验证截图；
- P0 无 TODO、假按钮、假图表、占位页面；
- README 包含启动、账号、架构、功能边界、演示脚本和故障排查。
